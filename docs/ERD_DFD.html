🔐 Thiết kế thu hồi khóa (Key Revocation): Khi user đổi cặp khóa RSA: (1) row cũ trong key_pairs được set status='revoked', revokedAt, revokedReason. (2) Row mới được tạo với status='active'. (3) messages.senderKeyID trỏ đến đúng phiên bản khóa đã ký → server dùng khóa đó để verify lịch sử dù hiện tại đã revoked — đảm bảo Non-repudiation. (4) contacts.cachedKeyID so với key_pairs WHERE userID=X AND status='active' → nếu khác nhau, hệ thống cảnh báo user kiểm tra danh tính (chống MITM khi key thay đổi đột ngột).
