Module 1 — User Authentication AUTH · P1 · SERVER

register(username, email, password)
Tạo tài khoản mới, hash mật khẩu, trả JWT và trigger sinh khóa RSA
P1 SERVER-SIDE
Input
usernamestring
Tên đăng nhập, UNIQUE trong DB
emailstring
Email, UNIQUE trong DB
passwordstring
Mật khẩu gốc — plaintext từ client
Output
userIDint
ID vừa tạo trong users table
sessionTokenstring
JWT signed, exp 24h
→ triggers generateKeyPair(userID)
Quá trình xử lý
1
Validate
Kiểm tra username và email chưa tồn tại trong DB. Nếu trùng → throw error ngay.
2
Hash mật khẩu
bcrypt.hash(password, rounds=12)
bcrypt tự sinh và nhúng salt 128-bit vào hash string — không cần lưu salt riêng.
3
Insert vào DB
Lưu user record vào users table với isVerified=false, status=offline.
4
Trigger sinh khóa RSA
Gọi async generateKeyPair(userID) — chạy ở client sau khi nhận response.
5
Phát JWT
JWT.sign({ userID, exp: 24h })
login(username, password)
Xác thực mật khẩu, trả JWT và encrypted PrivKey về client
P1 SERVER-SIDE
Input
usernamestring
passwordstring
Plaintext từ client
Output
sessionTokenstring (JWT)
encPrivKeystring
PrivKey đã AES-encrypt — server không đọc được
→ encPrivKey vào decryptPrivateKey() ở client
Quá trình xử lý
1
Lookup user
Tìm user theo username. Không tìm thấy → throw USER_NOT_FOUND.
2
Verify mật khẩu
bcrypt.verify(password, user.passwordHash)
bcrypt tự extract salt từ hash string để verify — không cần trường salt riêng.
3
Lấy encrypted PrivKey từ Key Store
Fetch key_pairs WHERE userID=X AND status='active'. Trả về privateKey — vẫn còn AES-encrypted, server không bao giờ giải mã.
4
Update trạng thái
Set status='online', lastSeen=NOW() trong users table.
5
Phát JWT + trả về

Module 2 — Key Management KEY MGMT · P2 · P3

generateKeyPair(userID, passphrase)
Sinh cặp khóa RSA-2048, mã hóa PrivKey bằng passphrase — chạy hoàn toàn ở client
P2 CLIENT-SIDE
Input
userIDint
passphrasestring
Nhập bởi user, dùng để bảo vệ PrivKey
Output
pubKeyB64string (Base64)
SPKI format — chia sẻ công khai được
encPrivKeystring
PrivKey đã AES-GCM encrypt
→ cả 2 vào storeKeyPair()
Quá trình xử lý
1
Sinh cặp khóa RSA
WebCrypto.generateKey — RSA-OAEP, 2048-bit, e=65537
Chạy trong browser sandbox, khóa chưa bao giờ ra khỏi client ở dạng plaintext.
2
Export Public Key
exportKey("spki") → Base64
SPKI là format chuẩn cho public key, an toàn để gửi lên server.
3
Derive AES key từ passphrase
PBKDF2(passphrase, salt=random16, iter=100000, SHA-256)
100,000 iterations để chống brute-force. Salt ngẫu nhiên đảm bảo cùng passphrase → AES key khác nhau.
4
Mã hóa Private Key
AES-256-GCM(aesKey, IV=random12, privKeyBytes)
GCM cung cấp auth tag — phát hiện ngay nếu ciphertext bị tamper khi giải mã. IV ngẫu nhiên 12 byte.
5
Upload lên server
Gọi storeKeyPair(). Server chỉ lưu bản đã encrypt, không thể đọc được nội dung khóa.
decryptPrivateKey(encPrivKey, passphrase)
Giải mã PrivKey từ server bằng passphrase — chạy ở client sau login
P2 CLIENT-SIDE
Input
encPrivKeystring
Nhận từ output của login()
passphrasestring
User nhập — dùng để re-derive AES key
Output
privateKeyCryptoKey
Nằm trong memory, không serialize, không gửi đi
→ vào encryptMessage() (ký) và decryptMessage() (giải mã)
Quá trình xử lý
1
Parse encPrivKey
Tách ra: { salt, IV, ciphertext, authTag } từ chuỗi Base64 đã lưu.
2
Re-derive AES key
PBKDF2(passphrase, salt, iter=100000)
Dùng đúng salt đã lưu → ra đúng AES key đã dùng lúc generateKeyPair.
3
Giải mã PrivKey
AES-256-GCM.decrypt(aesKey, IV, ciphertext)
GCM tự xác thực authTag — nếu passphrase sai hoặc data bị tamper → throw error ngay, không giải mã ra gì.
4
Import dưới dạng CryptoKey
importKey("pkcs8", extractable=false)
extractable=false nghĩa là key này không thể export ra bytes — chỉ dùng để sign/decrypt.
storeKeyPair(userID, pubKey, encPrivKey)
Lưu cặp khóa lên server (Key Store) — server chỉ lưu, không giải mã được PrivKey
P2 SERVER-SIDE
Input
userIDint
pubKeystring (Base64)
Public key — plaintext, an toàn để lưu
encPrivKeystring
Private key đã AES-encrypt
Output
keyIDint
ID của row trong key_pairs table
status"stored"
Quá trình xử lý
1
Verify JWT
Đảm bảo chỉ chính user đó mới upload key của mình — không ai upload thay được.
2
Insert key_pairs
Lưu với status='active', keySize=2048, algorithm='RSA-OAEP', createdAt=NOW().
getPublicKey(targetUserID)
Lấy Public Key của người nhận — kiểm tra cache và phát hiện key thay đổi đột ngột
P2 / P3 CLIENT-SIDE
Input
targetUserIDint
sessionTokenstring (JWT)
Output
pubKeyCryptoKey
Sẵn sàng dùng để RSA-OAEP encrypt
→ vào encryptMessage() làm receiverPubKey
Quá trình xử lý
1
Đọc cache từ contacts (Contact Store)
Lấy cachedPubKey và cachedKeyID của targetUserID trong contacts table.
2
Fetch key hiện tại từ server (Key Store)
Gọi API lấy key_pairs WHERE userID=X AND status='active'.
3
So sánh cache vs current
if cachedKeyID ≠ active keyID → ALERT
Key thay đổi đột ngột = dấu hiệu MITM. Cảnh báo user xác minh danh tính trước khi tiếp tục.
4
Cập nhật cache và import
Lưu key mới vào contacts.cachedPubKey và cachedKeyID. Import Base64 → CryptoKey để dùng.

Module 3 — Encryption Pipeline ENCRYPT & SIGN · P4 · CLIENT

encryptMessage(plaintext, receiverPubKey, senderPrivKey)
Hàm tổng hợp pipeline mã hóa — gọi 4 hàm con bên dưới theo thứ tự, trả về packet hoàn chỉnh
P4 · CORE CLIENT-SIDE
Input
plaintextstring
Nội dung tin nhắn gốc M
receiverPubKeyCryptoKey
Từ getPublicKey()
senderPrivKeyCryptoKey
Từ decryptPrivateKey(), trong memory
Output — packet gửi đi
Cstring (Base64)
Ciphertext — AES-256-CBC(M)
K'string (Base64)
RSA-OAEP(K) — session key đã mã hóa
Sstring (Base64)
RSA-PSS signature của H
Hstring (hex)
SHA-256(M)
IVstring (Base64)
AES IV — 16 bytes ngẫu nhiên
→ toàn bộ packet vào sendPacket()
Quá trình xử lý — gọi tuần tự 4 hàm con
1
H = hashPlaintext(plaintext)
Tính hash trước khi mã hóa để H phản ánh đúng M gốc.
2
K = random 256-bit · IV = random 16-byte
Session key và IV mới cho mỗi tin nhắn — perfect forward secrecy.
3
C = encryptAES(plaintext, K, IV)
4
K' = encryptRSA(K, receiverPubKey)
5
S = signHash(H, senderPrivKey)
6
Assemble packet
Return { C, K', S, H, IV }
hashPlaintext(plaintext)
Tính SHA-256 của tin nhắn gốc — đảm bảo Integrity
P4 · P6
Input
plaintextstring
Tin nhắn gốc M
Output
Hstring (hex 64 ký tự)
→ vào signHash() và lưu vào packet
→ tái sử dụng trong decryptMessage() để verify
Quá trình xử lý
1
SHA-256(encode(plaintext)) → hashBytes
Dùng WebCrypto crypto.subtle.digest("SHA-256").
2
Encode hashBytes → hex string 64 ký tự.
encryptAES(plaintext, K, IV)
Mã hóa nội dung bằng AES-256-CBC — Confidentiality của M
P4
Input
plaintextstring
KCryptoKey (256-bit)
Session key ngẫu nhiên
IVUint8Array (16 bytes)
Initialization vector ngẫu nhiên
Output
Cstring (Base64)
→ vào packet.C
Quá trình xử lý
1
AES-256-CBC.encrypt(K, IV, msgBytes)
Padding PKCS#7 tự động. Dùng AES vì nhanh hơn RSA nhiều với dữ liệu dài.
2
Encode cipherBuffer → Base64 string.
encryptRSA(K, receiverPubKey)
Mã hóa session key K bằng RSA-OAEP — chỉ receiver giải mã được K
P4
Input
KCryptoKey
AES session key 256-bit
receiverPubKeyCryptoKey
Public Key của người nhận
Output
K'string (Base64)
K đã mã hóa — chỉ PrivKey_B giải được
→ vào packet.K_enc
Quá trình xử lý
1
Export K dưới dạng raw bytes.
2
RSA-OAEP.encrypt(receiverPubKey, rawKey)
OAEP có padding ngẫu nhiên — cùng K → ciphertext khác nhau mỗi lần.
signHash(H, senderPrivKey)
Ký số hash bằng RSA-PSS — Authentication + Non-repudiation
P4
Input
Hstring (hex)
SHA-256 của plaintext M
senderPrivKeyCryptoKey
Private Key của người gửi
Output
Sstring (Base64)
Chữ ký RSA-PSS
→ vào packet.S · lưu vĩnh viễn trong messages table cùng senderKeyID
Quá trình xử lý
1
RSA-PSS.sign(senderPrivKey, hashBytes, saltLen=32)
PSS có salt ngẫu nhiên — cùng H và key → signature khác nhau mỗi lần (khác PKCS1v1.5).
2
Encode → Base64 string.

Module 4 — Transmit NETWORK · P5

sendPacket(packet, receiverID, token)
Lưu packet vào Message Store, deliver qua Socket nếu receiver online
P5 SERVER-SIDE
Input
packetobject
{C, K', S, H, IV} từ encryptMessage()
receiverIDint
tokenstring (JWT)
Để verify senderID
Output
messageIDint
ID trong messages table
status"sent" | "delivered"
→ packet đến receiver qua Socket event "new_message"
Quá trình xử lý
1
Verify JWT
Extract senderID từ token. Không ai gửi thay được.
2
Insert vào Message Store (messages table)
Lưu toàn bộ {C, K', S, H, IV} cùng senderKeyID, receiverKeyID, status='sent'. Server không đọc được nội dung.
3
Deliver qua Socket
Nếu receiver đang online → emit "new_message" event, update status='delivered'. Nếu offline → message chờ trong Message Store cho đến khi receiver fetch.
receivePacket(socketEvent)
Nhận Socket event, validate cấu trúc packet, chuyển sang pipeline giải mã
P5 CLIENT-SIDE
Input
socketEventSocket event
"new_message" từ server
 ↳ packet{C, K', S, H, IV}
 ↳ senderIDint
Output
packetobject (validated)
→ vào decryptMessage()
Quá trình xử lý
1
Structural validation
Assert đủ các field: C, K_enc, S, H, IV đều có và không rỗng.
2
ACK về server
Emit "ack" với messageID → server update status='delivered'.
3
Lấy Public Key của sender
Gọi getPublicKey(senderID) để dùng verify signature.
4
Chuyển sang decryptMessage()

Module 5 — Decryption Pipeline DECRYPT & VERIFY · P6 · CLIENT

decryptMessage(packet, receiverPrivKey, senderPubKey)
Hàm tổng hợp pipeline giải mã — gọi 3 hàm con, kiểm tra integrity và xác thực chữ ký, trả kết quả cuối
P6 · CORE CLIENT-SIDE
Input
packetobject
{C, K', S, H, IV} nhận từ socket hoặc load từ Message Store
receiverPrivKeyCryptoKey
Private Key của B — trong memory
senderPubKeyCryptoKey
Public Key của A — từ getPublicKey()
Output
plaintextstring | null
null nếu tamper detected
isVerifiedboolean
tamperAlertboolean
→ plaintext + isVerified vào UI render · tamperAlert=true → cảnh báo user, update DB
Quá trình xử lý — gọi tuần tự 3 hàm con + kiểm tra
1
K = decryptRSA(packet.K_enc, receiverPrivKey)
Khôi phục session key K từ K'.
2
M = decryptAES(packet.C, K, packet.IV)
Giải mã ra plaintext M.
3
H' = hashPlaintext(M)
Tính lại hash của M vừa giải mã được.
4
Kiểm tra integrity
if H' ≠ packet.H → tamperAlert = true
M bị thay đổi trên đường truyền → hash không khớp.
5
valid = verifySignature(packet.S, packet.H, senderPubKey)
if not valid → tamperAlert = true
Chữ ký không hợp lệ = không phải A gửi, hoặc S bị giả mạo.
6
Ghi kết quả vào DB
Update messages: sigVerified, tamperAlert, status='read'.
decryptRSA(K_enc, receiverPrivKey)
Giải mã session key K từ K' bằng RSA-OAEP
P6
Input
K_encstring (Base64)
Session key đã RSA-encrypt (K')
receiverPrivKeyCryptoKey
Output
KCryptoKey (AES 256-bit)
→ vào decryptAES()
Quá trình xử lý
1
RSA-OAEP.decrypt(receiverPrivKey, base64decode(K_enc))
2
Import rawKey → CryptoKey với algorithm AES-CBC, length 256.
decryptAES(C, K, IV)
Giải mã ciphertext bằng AES-256-CBC — khôi phục plaintext M
P6
Input
Cstring (Base64)
Ciphertext
KCryptoKey
Session key từ decryptRSA()
IVstring (Base64)
Đã lưu trong packet
Output
plaintextstring
Nội dung tin nhắn gốc M
→ vào hashPlaintext() để tính H'
Quá trình xử lý
1
AES-256-CBC.decrypt(K, IV, base64decode(C))
Unpadding PKCS#7 tự động. Nếu K hoặc IV sai → throw error.
2
TextDecoder → UTF-8 string.
verifySignature(S, H, senderPubKey)
Xác thực chữ ký RSA-PSS — Authentication + Non-repudiation
P6
Input
Sstring (Base64)
Chữ ký trong packet
Hstring (hex)
Hash trong packet
senderPubKeyCryptoKey
Public Key của sender A
Output
validboolean
false = giả mạo hoặc bị tamper
→ vào decryptMessage() để quyết định render hay alert
Quá trình xử lý
1
RSA-PSS.verify(senderPubKey, sigBytes, hashBytes)
WebCrypto trả về boolean. Verify dùng đúng keyID của lúc ký (senderKeyID) — hợp lệ ngay cả sau khi key bị revoke.

Module 6 — Message History HISTORY · P7

loadHistory(contactID, page, receiverPrivKey, senderPubKey)
Tải lịch sử từ Message Store, giải mã và xác thực từng packet — tái sử dụng decryptMessage()
P7 CLIENT-SIDE
Input
contactIDint
Lấy lịch sử hội thoại với contact này
pageint
Pagination — 20 tin mỗi trang
receiverPrivKeyCryptoKey
PrivKey của user đang đăng nhập
senderPubKeyCryptoKey
PubKey của contact để verify sig
Output
messages[]array
 ↳ msgIDint
 ↳ timestampstring
 ↳ plaintextstring | null
 ↳ isVerifiedboolean
 ↳ tamperAlertboolean
→ render chat UI với badge ✓ verified hoặc ⚠ tamper
Quá trình xử lý
1
Fetch packets từ Message Store
API GET /messages?contactID=X&page=N&limit=20&order=DESC. Server trả về mảng các packet {C, K', S, H, IV} — không giải mã.
2
Giải mã từng packet
for pkt of packets → decryptMessage(pkt, privKey, pubKey)
Tái sử dụng hoàn toàn pipeline P6. Mỗi tin nhắn được verify lại signature và integrity độc lập.
3
Xử lý kết quả
Nếu tamperAlert=true → hiển thị cảnh báo thay vì nội dung. Nếu isVerified=false → thêm badge ⚠ bên cạnh tin.
4
Return mảng đã giải mã
Render chat UI với đầy đủ thông tin xác thực cho mỗi tin nhắn.
